目标：
为背单词网站增加“读音来源选择”功能，用户可以选择不同发音来源，也可以使用自动模式。

读音来源：
1. 自动模式：按顺序尝试多个来源
2. 有道发音：优先用于国内访问
3. Free Dictionary：作为真实词典音频来源
4. 浏览器 TTS：作为兜底方案

默认策略：
默认使用“自动模式”。

自动模式调用顺序：
1. 先尝试有道发音
2. 如果播放失败，尝试 Free Dictionary API
3. 如果仍然失败，使用浏览器 SpeechSynthesis TTS
4. 如果全部失败，提示用户“当前无法播放读音，请稍后重试或切换读音来源”

用户设置：
1. 在设置页或听写页面增加“读音来源”选择项
2. 用户选择后保存到 localStorage
3. 下次进入页面时自动读取用户上次选择
4. 如果没有保存过，默认使用 auto

页面交互：
1. 用户点击播放按钮
2. 系统读取当前选择的读音来源
3. 根据来源调用对应的播放逻辑
4. 播放成功后无需打扰用户
5. 播放失败时显示轻量提示
6. 自动模式下可以在控制台记录实际使用的来源，方便调试

产品文案：
自动选择：推荐，系统会自动寻找可用读音
有道发音：国内访问较快，适合普通单词
Free Dictionary：词典音频，部分单词可能无音频
浏览器 TTS：稳定兜底，但声音取决于用户设备

注意事项：
1. 不要把有道直链作为唯一方案
2. Free Dictionary 不保证每个单词都有音频
3. 浏览器 TTS 在不同系统上的声音不同
4. iOS/Safari 可能要求用户点击后才能播放声音
5. 所有播放逻辑都必须由用户点击触发，避免浏览器拦截自动播放
6. 播放失败时不要让测试流程崩溃，只提示并允许继续

修复并完善听写页面的自动播放逻辑，同时增加音频启用提示。

背景：
当前听写页面缺少两个关键体验：
1. 进入听写流程后，第一个单词没有自动播放读音。
2. 进入下一个单词后，新单词没有自动播放读音。

目标：
1. 用户进入听写页面后，先显示一个“启用单词读音”的提示卡片。
2. 用户点击“开始听写并启用音频”后，正式开始测试，并立即播放第一个单词的读音。
3. 用户提交当前答案后，如果进入下一个单词，必须自动播放下一个单词的读音。
4. 用户点击“下一个单词”时，也必须自动播放新单词读音。
5. 用户重新开始测试、重试、手动切换单词时，也要播放当前单词读音。
6. 保留“再听一次”按钮，用户可以随时重复播放当前单词。

重要限制：
浏览器通常不允许网页刚打开就无用户操作自动播放音频，所以不要在页面加载完成后强行播放。
正确做法是：
用户点击“开始听写并启用音频”后，认为已经完成用户交互授权，然后播放第一个单词。
之后在同一次测试流程里，每次进入下一个单词，都自动触发播放。

音频提示卡片：
进入听写页面后，在测试开始前显示提示卡片。

推荐文案：
标题：启用单词读音

正文：
为了让系统在听写时自动播放每个单词的读音，请先点击下方按钮开启音频。部分浏览器可能会限制自动播放，如果没有声音，可以点击“再听一次”。

按钮：
开始听写并启用音频

交互流程：
1. 用户进入听写页面。
2. 显示“启用单词读音”提示卡片。
3. 测试未开始时，不自动播放任何音频。
4. 用户点击“开始听写并启用音频”。
5. 设置 isTestStarted = true。
6. 设置 isAudioUnlocked = true。
7. 立即调用当前单词的播放逻辑，播放第一个单词。
8. 后续 currentWordIndex 每次变化时，如果 isTestStarted 为 true，则自动播放新单词。
9. 用户点击“再听一次”时，重复播放当前单词。
10. 用户重新开始测试时，回到第一个单词并自动播放第一个单词。

播放时机：
1. 点击“开始听写并启用音频”后，播放当前第一个单词。
2. currentWordIndex 变化后，播放新单词。
3. 点击“下一个单词”后，播放新单词。
4. 答案提交成功并跳转到下一题后，播放新单词。
5. 重新开始测试后，播放第一个单词。
6. 手动切换单词后，播放切换后的当前单词。

读音来源：
使用现有 pronunciation provider 逻辑。
如果用户选择 auto，就按以下顺序尝试：
1. 有道发音
2. Free Dictionary
3. 浏览器 TTS

如果用户选择指定来源，就只使用对应来源。

状态设计：
需要考虑以下状态：
1. isTestStarted：测试是否已经开始。
2. isAudioUnlocked：用户是否已经完成音频交互授权。
3. isPlaying：当前是否正在播放音频。
4. autoPlayFailed：自动播放是否失败过。
5. currentWordIndex：当前单词索引。
6. selectedProvider：当前读音来源。

播放控制要求：
1. 增加 isPlaying 状态，防止连续点击导致多个音频重叠。
2. 播放新单词前，先停止上一个音频，或取消上一次 TTS。
3. 如果使用浏览器 TTS，播放前先调用 speechSynthesis.cancel()。
4. 如果使用 audio 对象，播放新音频前要暂停旧音频并重置。
5. 播放失败时不要中断测试流程。
6. 不要频繁弹窗，避免打断用户听写。

失败处理：
如果第一次播放失败：
1. 不要中断测试。
2. 设置 autoPlayFailed = true。
3. 在页面上显示轻量提示：
“浏览器可能阻止了自动播放，请点击再听一次，或在浏览器设置中允许本站播放声音。”
4. 保留“再听一次”按钮。
5. 允许用户继续答题。

如果后续自动播放失败：
1. 不要让页面报错。
2. 不要中断测试流程。
3. 显示轻量提示：
“读音播放失败，请点击再听一次或切换读音来源。”
4. 控制台记录失败来源，方便调试。

用户体验要求：
1. 页面进入后，用户能看到明确的音频启用提示。
2. 用户点击开始后，立即播放第一个单词。
3. 每次进入下一个单词时，自动播放一次。
4. 保留“再听一次”按钮。
5. 如果没有声音，用户知道应该点击“再听一次”或检查浏览器音频权限。
6. 不要用频繁 alert。
7. 不要让音频播放失败影响答题流程。

验收标准：
1. 进入听写页面后，不会立刻强行播放音频，而是显示“启用单词读音”提示。
2. 点击“开始听写并启用音频”后，可以听到第一个单词。
3. 答完一个单词并进入下一题后，自动播放新单词。
4. 点击“下一个单词”后，自动播放新单词。
5. 点击“再听一次”后，可以重复播放当前单词。
6. 重新开始测试后，可以自动播放第一个单词。
7. 自动模式下，有道失败后能继续尝试 Free Dictionary，再失败后使用浏览器 TTS。
8. 播放失败不会导致页面报错或测试中断。
9. 不会出现多个音频同时播放的问题。
10. 页面文案不要承诺浏览器一定会弹出权限窗口，因为不同浏览器策略不同。
