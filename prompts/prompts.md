我想开发一个背单词网站，请基于以下要求生成一个可运行的全栈项目。优先保证代码结构清晰、可维护、可扩展，不要只做静态页面。

项目目标：
做一个支持“听音拼写”的背单词网站。用户可以播放单词读音，根据音频输入英文单词拼写，系统判断正误，并记录学习结果。

技术要求：

- 前端：Next.js 15 + TypeScript + Tailwind CSS
- 路由：App Router
- 数据库：数据本地存储
- ORM：Drizzle ORM
- 认证：Better Auth
- 状态管理：优先使用 React 内置能力，必要时可用 Zustand
- 表单校验：Zod
- 不要使用 Redux
- 不要写 class 组件
- 尽量使用 Server Components，只有交互部分使用 Client Components

核心功能：

1. 用户系统

- 支持注册、登录、退出登录
- 每个用户只能看到和管理自己的词库与学习记录

1. 单词数据

- 系统内置默认单词列表，如list1,list2
- 用户可以自己导入单词和中文含义
- 导入格式至少支持手动输入
- 必须额外支持.txt导入，txt的格式为 英语单词:中文释义。单词之间用;隔开。
- **Free Dictionary API 做音频来源**
- 每个单词至少包含：
  - word
  - meaning
  - pronunciation_audio_url

1. 测试功能

- 测试时按单词列表抽取单词
- 单词乱序
- 播放单词音频
- 用户根据发音输入单词拼写
- 提交后显示正确或错误
- 错误时展示正确拼写和中文含义
- 记录本次答题结果

1. 学习记录

- 保存每次测试记录
- 至少包括：
  - user_id
  - word_id
  - user_answer
  - is_correct
  - answered_at
  - wordlist_id
- 页面上可以查看最近测试结果和错词

1. 音频能力

- 接入一个词典/发音 API，为单词提供可播放音频
- 如果单词没有音频，前端要有降级提示
- 音频播放按钮要可重复点击播放

页面需求：

- 首页：产品介绍 + 开始学习入口
- 登录/注册页
- 仪表盘页：显示词库数量、最近学习记录、错词数量
- 词库管理页：查看默认词库、我的词库、添加单词、导入单词
- 测试页：播放音频、输入拼写、提交答案、反馈结果、下一题
- 错词页：只显示答错过的单词

数据库设计：
请设计合理的数据表，并给出 Drizzle schema，至少包括：

- users
- word_lists
- words
- test_records

接口需求：
请实现必要的 API / server actions：

- 用户新增单词
- 用户导入单词
- 获取测试题
- 提交测试答案
- 获取最近学习记录
- 获取错词列表

UI 要求：

- 风格简洁、现代、适合学习类产品
- Tailwind 编写样式
- 响应式布局
- 输入框、按钮、卡片风格统一
- 测试页面要突出音频播放按钮和输入区

代码要求：

- 给出完整目录结构
- 给出关键文件代码
- 给出数据库 schema
- 给出初始化默认词库的 seed 脚本
- 给出环境变量示例
- 给出本地运行步骤
- 对关键代码加注释，但不要啰嗦
- 如果某些 API 需要第三方服务，请用可替换配置方式封装，不要把服务写死

额外要求：

- 先实现 MVP，保证可运行
- 代码优先可读性，不要过度抽象
- 如果你觉得某个功能实现有更优方案，请直接采用并说明原因
- 最后输出：
  1. 项目目录结构
  2. 数据库 schema
  3. 核心页面代码
  4. 核心接口代码
  5. 环境变量示例
  6. 启动步骤
