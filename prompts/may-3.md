请重构 admin 面板的页面结构。

当前问题：
- admin 面板把所有模块都直接堆在主页上
- 信息量太大，页面显得很乱
- 管理员进入页面后不容易快速找到自己想看的内容
- 缺少后台面板应有的模块化导航结构

目标：
将当前 admin 面板改成“左侧边栏 + 右侧内容区”的结构，让管理员可以通过侧边栏切换自己想查看的内容，而不是把所有统计和模块全部罗列在一个页面上。

具体要求如下：

1. 新增左侧边栏
请为 admin 面板增加一个左侧边栏，用来切换不同的数据模块。

边栏建议包含这些入口：
- 总览
- 用户分析
- 错误词分析
- 词库表现
- 最近活动
- 系统状态

如果你有更合理的命名，也可以优化，但整体方向就是把现有堆在主页上的内容拆成几个明确分类。

2. 右侧内容区按分类展示
- 点击左侧边栏的某个菜单项后，右侧只展示当前分类对应的内容
- 不要继续把所有模块同时显示在一个页面里
- 目标是让每个分类页面更聚焦、更清晰

例如：
- 总览：顶部统计卡片、关键总数据
- 用户分析：用户列表、用户正确率、练习量
- 错误词分析：全站错词 Top、近期错词排行
- 词库表现：词库使用人数、正确率、完成度
- 最近活动：用户最近行为流
- 系统状态：数据库状态、音频接口状态、证书状态等

3. 默认进入“总览”
- 进入 admin 面板时，默认展示“总览”
- 总览页不需要放全部详细数据，只保留最核心的全站概览信息

4. 页面风格要求
- 保持 admin 面板整体更像轻量后台，而不是学习页
- 但仍要和当前网站风格保持一致，不要突然变成完全不同的设计语言
- 边栏要清晰、简洁、有当前选中态
- 整体布局要比现在更有秩序感

5. 响应式要求
- 桌面端优先使用左侧边栏 + 右侧内容区
- 窄屏下边栏可以折叠、抽屉化，或者改为顶部切换
- 不要破坏移动端可用性

6. 交互要求
- 当前选中的菜单项要有明显高亮
- 切换不同模块时，内容区切换要清晰
- 不需要复杂动画，轻量即可
- 如果适合，可使用 query 参数、tab 状态或嵌套路由实现，但不要过度复杂

7. 目标效果
- admin 页面不再显得“所有内容都堆在一起”
- 管理员可以通过侧边栏快速选择自己要看的内容
- 页面更像一个真正可用的后台，而不是一个长长的数据展示页
