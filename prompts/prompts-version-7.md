请在当前项目中完成以下 Bug 修复和首页入场动画功能。请直接修改代码，不要只给建议。修改前先理解现有页面结构、组件拆分、状态逻辑和样式体系，尽量保持原有技术栈、设计风格和组件结构，不要大幅重构，不要引入新依赖，除非项目中已经使用了对应动画库。

一、Bug Fix：检查并修复词库分页功能

当前问题：
仪表盘/词库列表中的分页功能似乎没有成功实现。请检查现有分页代码是否真正生效，并修复相关问题。

期望行为：
1. 当词库数量大于 4 个时，每页只展示 4 个词库。
2. 当词库数量小于或等于 4 个时，不显示分页控件。
3. 分页控件需要包含：
   - 上一页
   - 下一页
   - 当前页码 / 总页数
4. 点击上一页、下一页后，词库卡片列表应实际切换，而不是只改变按钮状态。
5. 删除词库后需要重新计算页码：
   - 如果当前页还有数据，保持当前页；
   - 如果当前页已经没有数据，自动回退到上一页；
   - 页码不能小于 1，也不能大于总页数。
6. 新增词库、导入词库或删除词库后，分页状态不能异常。
7. 分页逻辑不能影响词库选择、编辑、删除、统计展示等原有功能。
8. 请检查是否存在以下常见错误：
   - 页面仍然渲染完整词库数组，而不是分页后的数组；
   - currentPage 状态没有被实际用于 slice；
   - totalPages 计算错误；
   - 删除词库后 currentPage 没有修正；
   - 分页组件渲染了，但按钮事件没有更新列表；
   - 异步加载词库后分页状态未同步。

二、Add Feature：首页入场动画

请为用户打开主页或刚刚进入网站时添加一段完整的首页入场动画。动画需要按顺序播放，总时长不超过 2 秒。

动画目标：
让首页首屏内容有一个品牌进入、标题浮现、介绍出现、操作按钮出现、学习流程卡牌出现的顺序感。

动画顺序如下：

1. Brand 入场
   - “克拉斯单词记忆”这个 brand 文本先在大屏幕中央展示。
   - 然后它一边缩小，一边移动到页面左上角，也就是最终导航栏 brand 的位置。
   - 动画应自然顺滑，不要突兀。
   - 注意：最终导航栏里的 brand 仍然要正常存在，不要造成重复闪烁或布局跳动。
   - 可以采用一个 overlay/intro layer 来承载入场 brand 动画，动画结束后隐藏 overlay。

2. 蓝色小标签出现
   - 在 brand 动画之后，显示“听音拼写背单词”这个蓝色小框框。
   - 需要有轻微的淡入或上移动画。

3. 主标题逐渐浮现
   - 显示文案：
     离开你的单词书！从音频出发练习听写
   - 需要渐显，可以配合轻微上移。

4. 介绍文案与主按钮出现
   - 显示介绍文案：
     克拉斯单词记忆 为每个用户隔离词库与学习记录，支持登录、导入单词、抽题测试、错词回顾，适合作为背单词网站的清晰 MVP 基线。
   - 同时或稍后显示两个按钮：
     进入仪表盘
     查看词库
   - 两个按钮需要有轻微的淡入、上移或缩放动画。

5. 学习流程卡牌出现
   - 最后显示学习流程相关的卡牌区域。
   - 卡牌可以按顺序轻微 stagger 出现。
   - 如果现有页面已经有学习流程卡牌，请只给它们增加入场动画，不要重写卡牌内容。

动画时间要求：
1. 总动画时长不超过 2 秒。
2. 建议时间分配：
   - brand 中央出现到移动左上角：0.6s 左右；
   - 蓝色小标签出现：0.2s 左右；
   - 主标题出现：0.3s 左右；
   - 介绍文案与按钮出现：0.3s 左右；
   - 学习流程卡牌出现：0.4s 左右。
3. 动画要有顺序感，但不能拖沓。
4. 动画结束后，页面必须可以正常交互。

实现要求：
1. 不要引入新依赖。
2. 优先使用 CSS animation、Tailwind transition/animation，或项目中已经存在的动画方案。
3. 如果项目是 Next.js App Router，请注意 Server Component / Client Component 边界：
   - 需要使用 useEffect、useState 或动画状态的组件必须是 Client Component；
   - 不要把整个页面无意义地改成 Client Component；
   - 可以把首页入场动画封装成单独的 Client Component。
4. 动画只在用户打开主页或刷新主页时播放一次。
5. 动画不应该阻塞页面加载太久。
6. 动画层不能在结束后遮挡按钮或影响点击。
7. 移动端也要适配：
   - 小屏幕下 brand 从屏幕中央移动到顶部左侧或导航栏位置；
   - 文案不能溢出；
   - 按钮和卡牌布局不能被动画破坏。
8. 如果用户系统设置了 prefers-reduced-motion: reduce，应尽量减少动画或直接展示最终状态。
9. 避免 hydration mismatch：
   - 不要依赖服务端和客户端不一致的随机值；
   - 初始状态要稳定；
   - 动画状态在客户端挂载后再切换。
