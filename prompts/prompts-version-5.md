 请根据下面的需求修改项目。注意：不要随意引入新依赖；不要破坏现有用户数据；涉及数据库结构变更时，要同步更新 schema、迁移、seed、导入逻辑、前端展示逻辑和相关类型定义。

一、Feature：词库与单词数据结构升级

1. 添加“删除词库”功能

目标：
用户可以在词库管理页面删除自己创建或导入的词库。

要求：
- 在词库管理页面为每个可删除词库增加“删除”入口。
- 删除前必须二次确认，避免误删。
- 删除词库时，需要同时处理该词库下的单词关联数据。
- 删除成功后，页面列表要立即刷新。
- 删除失败时，给出轻量错误提示。
- 默认词库是否允许删除，请根据当前项目逻辑判断；如果默认词库不适合删除，就禁止删除并给出提示。

2. 重构单词数据库结构

每个单词需要支持以下字段：

- 英语拼写
  - 必填。
  - 默写判题时，英式拼写和美式拼写都应该判对。
  - 展示时默认只展示美式拼写。

- 读音音频
  - 可选。
  - 用于播放系统音频。
  - 如果为空，不报错，播放时走现有 pronunciation provider 逻辑或展示为空。

- 多个中文意思
  - 必填至少一个。
  - 不要把多个中文意思混成一个长字符串。
  - 需要分别存储，方便后续展示、筛选和维护。

- 音标
  - 可选。
  - 单词领读时展示音标。
  - 如果没有音标，展示为空，不报错。

- 词性
  - 可选。
  - 使用常见缩写形式，例如：
    - 名词：n.
    - 动词：v.
    - 形容词：adj.
    - 副词：adv.
    - 介词：prep.
    - 连词：conj.
    - 代词：pron.
  - 如果没有词性，展示为空，不报错。

3. 英式/美式拼写判题规则

目标：
默写时，英式拼写和美式拼写都可以判对，但页面展示只展示美式拼写。

要求：
- 数据结构中需要能表达“展示拼写”和“可接受答案”。
- 建议主字段保存美式拼写，用于展示。
- 另设可接受答案列表或等价拼写列表，用于判题。
- 判题时，用户输入需要做基础规范化：
  - 去掉首尾空格。
  - 忽略大小写。
  - 根据项目需要，可考虑处理多余空格。
- 只要用户输入命中主拼写或可接受答案，就判对。

4. 重构现有默认词库

目标：
现有默认词库也要迁移到新的单词数据结构。

要求：
- 更新默认词库 seed 数据。
- 每个默认单词至少要有英语拼写和中文意思。
- 如果能补充音标、词性、音频字段，可以补充。
- 如果某些字段暂时没有，例如音频、音标、词性，不要阻塞导入，也不要导致页面报错。
- 确保默认词库在新结构下仍然可以正常测试、领读、复习。

5. 保留用户导入词库能力

目标：
用户原来的导入词库功能不能被破坏。

要求：
- 用户导入的词库仍然可以保存和使用。
- 导入时，英语拼写和中文意思是必填。
- 音频、音标、词性、英式拼写、可接受答案等字段都是可选。
- 可选字段不存在时，不报错，只在展示时为空。
- 旧格式导入仍尽量兼容。
- 如果用户导入的是简单格式，例如：
  word:中文;word:中文;
  仍然应该能导入成功。
- 对于多个中文意思，设计一种兼容方案：
  - 可以支持用中文逗号、英文逗号、顿号、分号等拆分。
  - 或者保留为一个意思，但内部仍用数组结构存储。
  - 将你设计的新格式导入方案告诉我并且给我一个example

二、Bug Fix：领读与错词本

1. 单词领读结束后没有反馈

问题：
当前单词领读完毕后，用户不知道已经结束。

要求：
- 领读全部完成后，页面需要有明确反馈。
- 可以显示轻量提示，例如：
  “领读完成”
  “本轮单词领读已结束”
- 不要使用频繁或打断体验的 alert。
- 可以使用页面内提示、toast 或状态文案。
- 结束后按钮状态要恢复，例如允许重新开始领读。

2. 非系统音频来源时，多遍领读只播放一遍

问题：
当使用的不是系统音频，而是其他读音来源时，即使设置多遍领读，也只播放一遍。

要求：
- 修复领读重复次数逻辑。
- 无论当前音频来源是：
  - 系统音频
  - 有道发音
  - Free Dictionary
  - 浏览器 TTS
  都必须遵守用户设置的领读遍数。
- 每一遍之间需要有合理间隔，避免声音重叠。
- 播放下一遍前，需要确保上一遍已经结束。
- 如果某一遍播放失败，不要中断整个领读流程，可以跳过或使用 fallback，但要记录 warning。

3. 错词本缺少单词音频来源选择按钮

问题：
错词本页面没有选择单词音频来源的入口。

要求：
- 在错词本页面增加读音来源选择功能。
- 选择项应复用现有 pronunciation provider 逻辑。
- 选项包括：
  - 自动选择
  - 有道发音
  - Free Dictionary
  - 浏览器 TTS
  - 如果项目已有“系统音频”，也加入系统音频
- 用户在错词本中选择的读音来源应与其他页面逻辑保持一致。
- 如果项目已有全局读音来源设置，优先复用全局设置，不要重复造状态。
